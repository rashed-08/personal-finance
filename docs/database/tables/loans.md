# Loans Table Specification

## Purpose

The `loans` table represents money borrowed from another person or money lent to another person.

A loan defines an ongoing financial relationship.

Actual money movement is recorded in the `transactions` table.

The loan itself contains metadata and current status.

# Business Rules

- A loan represents a financial agreement.
- Transactions record money movement.
- A loan may have multiple repayments.
- Closed loans cannot receive new repayment transactions.
- Historical loan records are immutable.
- Loans are never physically deleted.

---

# Outstanding Balance

Outstanding balance is calculated.

Formula

```
Original Amount

-

Total Repayments

=

Outstanding Balance
```

No outstanding balance column is stored.

# Design Decisions

- Loans contain metadata only.
- Repayment history comes from transactions.
- Outstanding balance is derived.
- Multiple loans per person are supported.
- Loan history is immutable.

---

# Final Statement

The `loans` table models financial obligations between the user and other parties.

It separates the concept of a loan from the financial events associated with it, ensuring a clean domain model, accurate reporting, and future compatibility with advanced repayment workflows.
