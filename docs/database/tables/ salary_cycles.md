# Salary Cycles Table Specification

## Purpose

The `salary_cycles` table represents a financial reporting period based on the user's salary cycle rather than the calendar month.

A salary cycle groups transactions for reporting, carry-forward calculations, budgeting, and financial analysis.

Version 1 supports one active salary cycle configuration.

# Business Rules

- Every financial transaction belongs to exactly one salary cycle.
- Salary cycles must not overlap.
- Only one salary cycle may be OPEN at a time.
- Closed salary cycles are immutable.
- Reports are generated from salary cycle membership.
- Calendar month and salary cycle are independent concepts.

---

# Carry Forward

Carry Forward is **not stored** in this table.

It is calculated from transaction history.

Formula

```
Opening Balance

+

Income

-

Expense

=

Carry Forward
```

Future versions may materialize carry-forward for performance.

# Design Decisions

- Salary cycles are independent of calendar months.
- Carry-forward is calculated, not stored.
- Historical cycles remain immutable.
- Salary received date may differ from expected salary date.
- Reports are generated from cycle membership rather than transaction dates alone.

---

# Final Statement

The `salary_cycles` table provides the temporal backbone of the Personal Finance App.

It allows financial reporting to align with the user's real-world salary schedule instead of calendar months, enabling accurate carry-forward calculations, spending analysis, and long-term financial tracking.
